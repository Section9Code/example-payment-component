01

Overview

<payment-form> is a custom element that renders a card entry form, validates input locally, and handles the payment submission end-to-end. It knows nothing about the host page. The host page knows nothing about card entry.

Communication runs one way in (via a configure() method) and one way out (via DOM events that bubble out of the shadow root).

Why it's shaped this way Card entry and payment submission live inside the shadow DOM; the host only decides what to do when a payment succeeds, fails, or is reset. The component owns the gateway integration — you never see the card details.
02

Reference the component

The component registers itself as <payment-form> the first time its module is evaluated. Load it once — anywhere on the page or at the top of your app bundle.

import 
// Option A — classic script tag in HTML
<script type="module" src="https://pay.ej.com/payment-form.js"></script>

// Option B — ES module import from your entry file
import 'https://pay.ej.com/payment-form.js';
// Import once, typically at the top of your app entry.
// The module self-registers the <payment-form> element.
import 'https://pay.ej.com/payment-form.js';
Side-effect import You do not import the class. Just loading the file calls customElements.define('payment-form', ...). Guard against double-registration if your bundler eagerly re-evaluates modules.
03

Put it on the page

Drop the tag anywhere a block-level element is allowed. It sizes to its own shadow layout — you can wrap it in a container to constrain width.

markup 
<main>
  <payment-form id="checkout"></payment-form>
</main>
import { useRef } from 'react';

export function Checkout() {
  const formRef = useRef(null);

  return (
    <main>
      <payment-form ref={formRef} />
    </main>
  );
}
React Hold a ref — you'll need it to call configure() and attach listeners. React 19 supports custom elements natively; earlier versions render the tag fine but cannot pass non-string props or bind custom events via JSX, so the ref approach is the portable path.
04

Configure it

Call configure() before the user interacts with the form. A partial object is fine — omitted keys keep their defaults. Unknown keys and invalid values are silently ignored. You can re-call at any time; the form re-renders in place.

Key Type Default Description
submitLabel string "Pay Now" Primary button text. Amount is appended when amount + currency are set.
cancelLabel string "Cancel" Secondary button text (fires payment-reset).
heading string? — Renders an <h2> inside the form. Omit for no heading.
requireCvv boolean true When false, the CVV field is removed from the DOM and omitted from validation & processor details.
primaryColor css-color "#3b82f6" Applied as --pf-primary; drives submit background and focus outlines.
amount number — Included in processor details. Paired with currency, formats the submit label.
currency ISO 4217 — Used with amount via Intl.NumberFormat for display.
configure 
const form = document.querySelector('payment-form');

form.configure({
  heading: 'Complete your booking',
  submitLabel: 'Book flight',
  primaryColor: '#ff6600',
  amount: 199.00,
  currency: 'GBP',
});

// Re-call any time to update live:
form.configure({ amount: 3599 });
import { useEffect, useRef } from 'react';

export function Checkout({ amount }) {
  const formRef = useRef(null);

  useEffect(() => {
    formRef.current?.configure({
      heading: 'Complete your booking',
      submitLabel: 'Book flight',
      primaryColor: '#ff6600',
      amount,
      currency: 'GBP',
    });
  }, [amount]);

  return <payment-form ref={formRef} />;
}
05

Wire up the events

The component tells the host page what happened via three DOM events that bubble out of the shadow root. All three are composed, so you can listen on the element itself or on any ancestor in light DOM.

  • payment-success The payment was approved. No detail.
  • payment-error The payment was declined or failed. detail.message holds the message shown to the user.
  • payment-reset The user clicked the cancel button; the form has returned to its idle state. No detail.
events 
form.addEventListener('payment-success', () => {
  navigate('/thanks');
});

form.addEventListener('payment-error', (e) => {
  toast.error(e.detail.message);
});

form.addEventListener('payment-reset', () => {
  analytics.track('checkout_abandoned');
});
useEffect(() => {
  const el = formRef.current;
  if (!el) return;

  const onSuccess = () => navigate('/thanks');
  const onError = (e) => toast.error(e.detail.message);
  const onReset = () => analytics.track('checkout_abandoned');

  el.addEventListener('payment-success', onSuccess);
  el.addEventListener('payment-error', onError);
  el.addEventListener('payment-reset', onReset);
  return () => {
    el.removeEventListener('payment-success', onSuccess);
    el.removeEventListener('payment-error', onError);
    el.removeEventListener('payment-reset', onReset);
  };
}, []);

06

Putting it together

A minimal working host — configure once, attach listeners, react to outcomes. The component handles card entry, validation, and the bank call itself.

checkout 
import 'https://pay.ej.com/payment-form.js';

const form = document.querySelector('payment-form');

form.configure({
  heading: 'Complete your booking',
  submitLabel: 'Book flight',
  primaryColor: '#ff6600',
  amount: 199,
  currency: 'GBP',
});

form.addEventListener('payment-success', () => navigate('/thanks'));
form.addEventListener('payment-error', (e) => toast.error(e.detail.message));
import { useEffect, useRef } from 'react';
import 'https://pay.ej.com/payment-form.js';

export function Checkout({ amount, onSuccess, onError }) {
  const formRef = useRef(null);

  useEffect(() => {
    const el = formRef.current;
    if (!el) return;

    el.configure({
      heading: 'Complete your booking',
      submitLabel: 'Book flight',
      primaryColor: '#ff6600',
      amount,
      currency: 'GBP',
    });

    el.addEventListener('payment-success', onSuccess);
    el.addEventListener('payment-error', onError);
    return () => {
      el.removeEventListener('payment-success', onSuccess);
      el.removeEventListener('payment-error', onError);
    };
  }, [amount, onSuccess, onError]);

  return <payment-form ref={formRef} />;
}
